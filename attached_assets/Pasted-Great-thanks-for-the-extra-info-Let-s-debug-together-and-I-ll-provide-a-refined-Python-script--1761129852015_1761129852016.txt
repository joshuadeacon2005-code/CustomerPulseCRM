## ✅ What to verify first

Here are common pitfalls when retrieving to-dos via the Basecamp 4 API:

1. **Correct API version / base URL**

   * You need to prefix with your account ID: e.g. `https://3.basecampapi.com/{ACCOUNT_ID}/...` as shown in the docs. ([GitHub][1])
   * Ensure you’re not accidentally pointing to the wrong version (Basecamp 3 vs Basecamp 4) or using the wrong endpoints.

2. **Correct endpoint path for to-dos**

   * First you need to identify the *to-do list(s)* (sometimes called “todolists”) within a project. For example:
     `GET /buckets/{project_id}/todosets/{todoset_id}/todolists.json` to list the lists. ([GitHub][2])
   * Then you fetch to-dos: `GET /buckets/{project_id}/todolists/{todolist_id}/todos.json` (for active/pending to-dos). ([GitHub][3])
   * If you skip the “todosets” layer (depending on how your instance is structured) you might not get results.

3. **Check for pagination / filters**

   * The API is paginated (via `Link` header) and may only return a subset by default. If you’re only looking for first page and it’s empty (e.g., no active tasks) it may appear as “no to-dos”. ([GitHub][1])
   * Also: some endpoints only return *active/pending* to-dos unless you include query parameters. ([GitHub][3])
   * Example: if all your to-dos are completed/archived, you might not see them unless you query for them.

4. **Permissions / scope**

   * Ensure the OAuth token you’re using has the correct scope to read to-dos and lists.
   * Ensure the user/account tied to the token has access to the projects + lists in question.

5. **Mapping & expectation in your code / CRM import**

   * Double-check that your code is looping through all levels: project → todoset → todolist → todos.
   * Check logs: maybe the lists are fetched but they are empty or your code filters them out.
   * Print/debug what your API returns (lists and items) to verify there *are* items but maybe you’re mis-mapping.

6. **Check for “empty” lists**

   * It’s possible you have projects and to-do lists defined, but no items inside those lists. So the lists endpoint returns a list of lists, but each list may have zero todos. Your code should log each list’s `id` and the count of items fetched so you can see if it’s “0”.

## 🐍 Python version of the import script

Here’s a **starter Python script** you can run in Replit (or locally) which uses the Basecamp 4 API, fetches to-do lists and items, and then prints them (you can adapt it to import into your CRM). Be sure to replace placeholders with your actual environment variables.

```python
import os
import requests

ACCESS_TOKEN = os.getenv("BASECAMP_ACCESS_TOKEN")
ACCOUNT_ID   = os.getenv("BASECAMP_ACCOUNT_ID")
CRM_API_URL  = os.getenv("CRM_API_URL")  # e.g., your Replit CRM endpoint

USER_AGENT = "MyCRMIntegration (you@example.com)"

BASE_URL = f"https://3.basecampapi.com/{ACCOUNT_ID}"

headers = {
    "Authorization": f"Bearer {ACCESS_TOKEN}",
    "User-Agent":   USER_AGENT,
    "Accept":       "application/json"
}

def get_json(url):
    resp = requests.get(url, headers=headers)
    if not resp.ok:
        raise Exception(f"Request failed: {url} --> {resp.status_code} {resp.text}")
    return resp.json(), resp.headers

def get_projects():
    url = f"{BASE_URL}/projects.json"
    data, _ = get_json(url)
    return data

def get_todo_sets(project_id):
    url = f"{BASE_URL}/buckets/{project_id}/todosets.json"
    data, _ = get_json(url)
    return data

def get_todo_lists(project_id, todoset_id):
    url = f"{BASE_URL}/buckets/{project_id}/todosets/{todoset_id}/todolists.json"
    data, _ = get_json(url)
    return data

def get_todo_items(project_id, todolist_id):
    url = f"{BASE_URL}/buckets/{project_id}/todolists/{todolist_id}/todos.json"
    data, _ = get_json(url)
    return data

def import_to_crm(task):
    # Example: POST to CRM endpoint
    resp = requests.post(CRM_API_URL, json=task)
    if not resp.ok:
        print(f"CRM import failed for task {task.get('title')}: {resp.status_code} {resp.text}")

def main():
    projects = get_projects()
    print("Found projects:", len(projects))
    for proj in projects:
        pid = proj.get("id")
        name = proj.get("name")
        print(f"\nProject {pid}: {name}")
        todo_sets = get_todo_sets(pid)
        if not todo_sets:
            print("  → No todo sets found")
        for ts in todo_sets:
            ts_id = ts.get("id")
            ts_title = ts.get("title")
            print(f"  TodoSet {ts_id}: {ts_title}")
            lists = get_todo_lists(pid, ts_id)
            if not lists:
                print("    → No to-do lists found in this set")
            for lst in lists:
                lst_id = lst.get("id")
                lst_title = lst.get("title")
                print(f"    List {lst_id}: {lst_title}")
                items = get_todo_items(pid, lst_id)
                print(f"      -> Found {len(items)} items")
                for it in items:
                    task = {
                        "projectId":   pid,
                        "projectName": name,
                        "listId":      lst_id,
                        "listTitle":   lst_title,
                        "todoId":      it.get("id"),
                        "title":       it.get("content"),
                        "description": it.get("description"),
                        "dueDate":     it.get("due_on"),
                        "completed":   it.get("completed"),
                        "assignees":   it.get("assigned_to_ids"),
                    }
                    print("        Task:", task["title"])
                    # Uncomment if you want to import:
                    # import_to_crm(task)

if __name__ == "__main__":
    main()
```

### 🧩 Notes on this script

* It prints counts of projects, to-do sets, lists, and items so you can see where things “stop”.
* If you see “0 items” for each list, then the issue is upstream (no data to fetch) rather than your mapping.
* If you see lists but your CRM still shows none, then the import step or mapping is likely failing.
* You can add debug/logging around each step (e.g., print response JSON for a sample list) to inspect fields.
* After verifying, you can adapt `import_to_crm` to match your CRM’s API schema.

## 🔍 How to proceed and debug further

1. Run the Python script with **print only** (no import yet) to see the full tree: project → todoset → lists → items.
2. If you get projects and todo-sets + lists but *0 items*, then:

   * Check in the Basecamp web UI if the project actually has to-do items (and they are pending/active).
   * Check if items are completed/archived and thus omitted by default; you may need to add a query parameter to include completed items.
3. If you get no todo-sets or lists:

   * Possibly the project has no “to-do tool” enabled or the project is of a type that doesn’t support to-dos.
   * Confirm you’re using the correct project ID / bucket ID.
4. If you get items but the CRM shows none: inspect your `import_to_crm` call:

   * Are you getting errors? Log the HTTP response for failures.
   * Is your CRM endpoint reachable from Replit (network, CORS, auth)?
   * Are you mapping fields correctly (e.g., required fields missing causing rejection)?
5. Consider adding **pagination** logic in your script in case you have many items and only the first page is returned.
6. Lastly, check the logs for HTTP status codes from Basecamp (e.g., 401 unauthorized, 403 forbidden, 404 endpoint not found) which might hint at the problem.
