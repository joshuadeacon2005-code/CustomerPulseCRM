Goal
Build a web-based sales management app to track customers, leads, interactions and sales, with role-based access for three user levels (global/regional, country manager, individual salesperson). Provide dashboards, CRUD flows, reporting by time/brand/team, and a robust backend.

Important Basecamp integration note

The app should import to-dos/tasks from my Basecamp account into the CRM (Basecamp → CRM). This import is currently not working as intended and must be implemented or debugged.
Ensure the integration supports: incremental sync, mapping Basecamp project/to-do fields to CRM tasks, handling attachments/comments, and conflict resolution.
Provide clear logs and error handling for the sync process, and an admin UI to trigger manual syncs, view last-sync status, and retry failed items.
Security: use OAuth or API tokens securely; do not expose Basecamp credentials. Respect rate limits and implement exponential backoff.
Audit: record sync operations (timestamp, user/admin, items imported, errors).
Acceptance: demonstrate a successful sync of recent Basecamp to-dos into CRM with sample data.
User Roles & Permissions

Level 1 (CEO / Sales MKTG Director / Regional Sales Director)
View data for all countries or selected countries.
Select countries and drill into any salesperson.
View customer profiles, monthly/quarterly/year targets and actuals by country and person.
Level 2 (Country Sales Manager)
Access only their country/team data.
View team and individual targets vs actuals, brand performance.
Switch view to any team member’s homepage.
Level 3 (Individual Salesperson)
Access only their own customer interactions, leads, sales and related data.
Personal homepage with quick actions and personalized stats.

Data, Backend & Security

Database: PostgreSQL recommended (or MySQL). Normalize customers, contacts, users, teams, countries, leads, interactions, sales, targets, tasks.
Role-based access control (RBAC): enforce per-endpoint and per-query filters (country/team scoping).
Validation & error handling: server- and client-side validation, clear error messages.
Audit/logging: record create/update/delete for customer, interaction, sale, lead (user, timestamp).
Multi-country data partitioning: include country_id on customer, sale, interaction, lead for easy filtering.
Basecamp integration specifics:
Implement a reliable import pipeline (webhooks if available, plus scheduled incremental sync).
Map Basecamp fields (project, to-do list, to-do item, assignee, due date, description, comments, attachments) to CRM task model.
Ensure idempotency and conflict resolution (e.g., track Basecamp IDs, update rather than duplicate).
Provide admin controls: manual sync trigger, view last sync time, retry failed imports, and clear error messages/logs.
Secure authentication using OAuth2 or secure API tokens; store secrets in environment variables / secret manager.
Rate limiting and exponential backoff for API errors.
Logging and metrics for monitoring sync health.
UI / UX

Clean responsive UI with fast quick-action buttons on homepages.
Dashboard widgets: KPIs (target vs actual), charts (time series, brand breakdown), recent items, tasks.
Filters: date range, country, salesperson, brand, retailer type.
Mobile-friendly forms for field reps.
Reporting & Filters

Sales by period (month/quarter/year), by brand, by country, by salesperson.
Export CSV/XLS for filtered results.
Calendar view for historical/future targets and interactions.
