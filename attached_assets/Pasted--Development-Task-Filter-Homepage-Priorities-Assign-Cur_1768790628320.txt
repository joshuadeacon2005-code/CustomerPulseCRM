# Development Task: Filter Homepage Priorities & Assign Currency by Region/Salesperson

## Context
This is a B2B CRM system for sales teams across multiple regional offices (Hong Kong, Singapore, Shanghai, Australia, New Zealand, Indonesia, Malaysia, Guangzhou). The app currently shows all action items and has a currency selector that users can manually change.

## Feature 1: Filter Homepage Priorities to Show Only User's Own Tasks

### Current Problem
The dashboard at `/client/src/pages/dashboard.tsx` currently displays action items from ALL customers. Lines 152-155 show:
```typescript
const userActionItems = actionItems.filter(item => 
  userCustomerIds.includes(item.customerId)
);
```
This filters by customers assigned to the user, but the action items themselves may have been created by other team members for those customers.

### Required Changes

#### Backend Changes
No backend changes needed - the filtering logic is already set up to use `createdBy` field from the `actionItems` table (line 152 in schema.ts).

#### Frontend Changes (`/client/src/pages/dashboard.tsx`)

**Location: Lines 152-155**

**Current Code:**
```typescript
const userActionItems = actionItems.filter(item => 
  userCustomerIds.includes(item.customerId)
);
```

**Updated Code:**
```typescript
// Filter to show ONLY the logged-in user's own action items
const userActionItems = actionItems.filter(item => 
  item.createdBy === effectiveUserId
);
```

**Impact:**
- This ensures that `overdueTasks`, `todayTasks`, and `upcomingTasks` (lines 158-160) only show action items created BY the logged-in user
- Removes customer action items and other team members' to-dos from the priorities section
- The filtering applies to:
  - Priority Alerts section (lines 271-344)
  - Upcoming Tasks section (lines 469-580)
  - Calendar view events

### Required Changes

#### Step 1: Define Regional Office to Currency Mapping

**Create new file: `/shared/currency-mapping.ts`**

```typescript
import type { Currency } from "./schema";

export const REGIONAL_OFFICE_CURRENCY_MAP: Record<string, Currency> = {
  "Hong Kong": "HKD",
  "Singapore": "SGD",
  "Shanghai": "CNY",
  "Australia": "AUD",
  "New Zealand": "NZD",
  "Indonesia": "IDR",
  "Malaysia": "MYR",
  "Guangzhou": "CNY",
} as const;

/**
 * Get the default currency for a regional office
 */
export function getCurrencyForRegionalOffice(regionalOffice: string | null | undefined): Currency {
  if (!regionalOffice) return "USD";
  return REGIONAL_OFFICE_CURRENCY_MAP[regionalOffice] || "USD";
}
```

## Migration Strategy (Important!)

### For Existing Users Without regionalOffice Set

**Create migration script: `/server/migrate-user-currencies.ts`**

```typescript
import { db } from "./db";
import { users } from "@shared/schema";
import { getCurrencyForRegionalOffice } from "@shared/currency-mapping";
import { eq } from "drizzle-orm";

async function migrateUserCurrencies() {
  const allUsers = await db.select().from(users);
  
  for (const user of allUsers) {
    const newCurrency = getCurrencyForRegionalOffice(user.regionalOffice);
    
    // Only update if different from current
    if (user.preferredCurrency !== newCurrency) {
      await db
        .update(users)
        .set({ preferredCurrency: newCurrency })
        .where(eq(users.id, user.id));
      
      console.log(`Updated ${user.username}: ${user.preferredCurrency} -> ${newCurrency}`);
    }
  }
  
  console.log("Migration complete!");
}

migrateUserCurrencies();
```

**Run this once after deploying:**
```bash
npx tsx server/migrate-user-currencies.ts
```
