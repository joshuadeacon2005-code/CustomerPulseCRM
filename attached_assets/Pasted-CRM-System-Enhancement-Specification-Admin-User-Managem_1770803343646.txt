CRM System Enhancement Specification - Admin User Management Dashboard
CONTEXT
You are working on an existing React/TypeScript CRM system with PostgreSQL backend hosted on Replit. The system has users (sales representatives), customers, targets, and sales records. You need to enhance the admin dashboard's user management capabilities.
OBJECTIVE
Implement a comprehensive admin view for individual users that consolidates all user-related data, analytics, and navigation shortcuts in one interface.
REQUIREMENTS - IMPLEMENT EACH POINT COMPLETELY
1. ADMIN USER PROFILE PAGE ENHANCEMENTS
1.1 User Dashboard Display

When admin clicks on any user from the user list, display that user's complete dashboard
Show ALL dashboard widgets and views that the user themselves would see
Render user's personal dashboard in read-only mode within admin interface

1.2 User Targets Display

Create dedicated "Targets" section visible on user profile page
Display all personal targets assigned to this user
Show target details: target amount, time period, product/category, progress percentage, deadline
Include visual progress indicators (progress bars or completion percentages)

1.3 User Sales History

Add "Sales History" section showing complete chronological sales records
Display: date, customer name (clickable), product/service, amount, status
Include filters: date range, customer, product, status
Show summary metrics: total sales, average deal size, conversion rate

2. CUSTOMER NAVIGATION - DIRECT LINKING
2.1 Universal Customer Name Click Behavior

RULE: Wherever a customer name appears in the admin interface, clicking it MUST redirect directly to that customer's profile page
Apply to ALL locations: user dashboard, customer lists, sales records, target views, analytics tables, kanban cards
Implementation: Wrap all customer name text in Link/anchor tags pointing to /admin/customers/{customerId} or equivalent route

2.2 Remove Intermediate Pages

ELIMINATE redirection to customer search page when clicking customer names
Direct navigation: Click customer name → Immediate customer profile load
Preserve back-navigation functionality to return to previous admin view

3. ADMIN USER ANALYTICS SECTION
3.1 Comprehensive Analytics Display

Add "Analytics" section to admin user profile page
Include metrics:

Total sales value (current month, quarter, year)
Number of deals closed
Conversion rate (opportunities → closed deals)
Average deal size
Sales velocity (average time to close)
Target achievement percentage
Activity metrics (calls, meetings, emails if tracked)



3.2 Visual Representations

Render charts: line graphs for sales trends, bar charts for comparisons, pie charts for category breakdowns
Make time period selectable (week, month, quarter, year)
Show year-over-year or period-over-period comparisons

4. CUSTOMER KANBAN PIPELINE VIEW
4.1 Kanban Board Implementation

Create kanban-style board showing ALL user's customers organized by pipeline stage
Stages (customize to your CRM): "Lead" → "Qualified" → "Proposal" → "Negotiation" → "Closed Won" / "Closed Lost"
Each card shows: customer name (clickable), deal value, last contact date, next action

4.2 Kanban Functionality

Cards are draggable between stages (optional, or read-only for admin)
Click customer name on any card → Direct navigation to customer profile
Display count of customers in each stage at column header
Filter/search functionality within kanban view

5. EXPANDED TARGET VIEW
5.1 Comprehensive Target-Customer Mapping

Create "Targets & Customers" view accessible from user profile
Display structure:

  Target 1: [Target Name] - $[Amount] - [Time Period]
    ├─ Customer A (Target: $X, Achieved: $Y)
    ├─ Customer B (Target: $X, Achieved: $Y)
    └─ Customer C (No target assigned)
  
  Target 2: [Target Name] - $[Amount] - [Time Period]
    ├─ Customer D (Target: $X, Achieved: $Y)
5.2 Show ALL Customers

Include customers WITH assigned targets (show target amount and progress)
Include customers WITHOUT assigned targets (display "No target assigned")
Make customer names clickable → Direct navigation to customer profile

5.3 Sales Button - Three-Column View

Add "View Sales Breakdown" button/toggle
When clicked, transform view to three-column table:

Column 1: CUSTOMER (name, clickable)
Column 2: TARGET (assigned target amount or "None")
Column 3: SALES (actual sales amount, variance from target if applicable)


Sort options: by customer name, target amount, sales amount, variance
Click customer name → Direct navigation to customer profile

6. DATABASE QUERIES REQUIRED
6.1 User Data Aggregation Query
sql-- Fetch all data needed for admin user profile page in single query or optimized multi-query
SELECT 
  users.*, 
  targets.*,
  customers.*,
  sales.*,
  -- Aggregate analytics
FROM users
LEFT JOIN targets ON users.id = targets.user_id
LEFT JOIN customers ON users.id = customers.assigned_user_id
LEFT JOIN sales ON customers.id = sales.customer_id
WHERE users.id = [USER_ID]
6.2 Performance Optimization

Index customer names, user IDs, and foreign keys
Implement pagination for long customer lists (100+ customers)
Cache analytics calculations when possible

7. UI/UX SPECIFICATIONS
7.1 Layout Structure

Use tabbed interface or expandable sections for different views: "Dashboard" | "Targets" | "Sales History" | "Analytics" | "Customer Pipeline"
Maintain consistent admin navigation header
Breadcrumb trail: Admin → Users → [User Name]

7.2 Clickable Elements Styling

Customer names: Styled as links (colored, underlined on hover, cursor pointer)
Consistent styling across all customer name instances
Hover state shows tooltip with quick info (last contact, deal stage)

8. ROUTING CONFIGURATION
8.1 Route Definitions

/admin/users/:userId - Main admin user profile page (implement all above features here)
/admin/customers/:customerId - Customer profile page (target of all customer name clicks)
Ensure proper route parameter passing and state preservation
