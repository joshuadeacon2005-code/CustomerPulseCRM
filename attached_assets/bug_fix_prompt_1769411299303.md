# Bug Fix: Target Display Issues

## User-Reported Issues

### Issue 1: Data Discrepancy in "Current Selection" Panel
**User Report:** "My Feb is 56840, but somehow there is a different figure shown above"
- User enters "56840.00" in the "Edit Personal Target for Feb" input
- The "Current Selection" panel shows "$60200.00" instead
- This is a critical data integrity issue

### Issue 2: Incomplete Table Display
**User Report:** "I have also input targets for 12 months but we can only see up to June"
- User has entered targets for all 12 months (Jan-Dec 2026)
- The "All Personal Targets for 2026" table only displays Jan-Jun
- Jul-Dec targets are hidden despite being saved in the database

### Issue 3: Actual Sales Not Updating on Dashboard
**User Report:** "This figure also didn't update according to the transactions I just logged"
- User logs new sales transactions
- Dashboard "My Personal Targets" card shows Jan 2026: $36,870 / $64,960 (57%)
- The $36,870 "actual sales" figure does NOT update after logging new transactions
- This is a **critical data sync issue** between the `sales` table and `monthly_sales_tracking` table

## Root Cause Analysis

### Issue 1: Currency Conversion Bug

**Location:** `/server/routes.ts` - GET `/api/targets` endpoint

**Problem:** The app likely has currency conversion logic that's converting stored values to the user's preferred currency. However, there are several potential causes:

1. **Race Condition**: When the user types in the input field, the form shows their input (56840), but the "Current Selection" panel is displaying data from the database query that may have:
   - An old cached value
   - A value that was saved in a different currency and is being converted
   - A value that was incorrectly saved due to currency conversion on save

2. **Double Conversion**: The value might be getting converted twice:
   - Once when saving (converts user input to base currency)
   - Again when retrieving (converts base currency to user's preferred currency)
   - This would cause: 56840 HKD → ~7300 USD (base) → 60200 HKD (if reconverted with wrong rate)

3. **State Management Issue**: The React component is showing:
   - Form input: Direct user input (56840)
   - Current Selection panel: Database value from React Query cache
   - These are out of sync

**Files to Investigate:**
- `/server/routes.ts` - Lines around GET `/api/targets` and PATCH `/api/targets/:id`
- `/client/src/pages/targets.tsx` - Lines 69-76 (selectedMonthTarget useMemo)
- Check if there's currency conversion happening in the GET endpoint
- Check if the PATCH endpoint is correctly handling currency conversion

### Issue 2: Table Scroll/Display Bug

**Location:** `/client/src/pages/targets.tsx` - Lines 360-414

**Problem:** The table container likely has CSS issues preventing scrolling or full display:

1. **Fixed Height Container**: The table or its parent may have a fixed height with `overflow: hidden` instead of `overflow-y: auto`
2. **Missing Scroll Container**: The `overflow-x-auto` class on line 360 only handles horizontal scrolling
3. **Viewport Issue**: The table height might be constrained by parent container dimensions

**Current Code:**
```tsx
<div className="overflow-x-auto">
  <table className="w-full" data-testid="table-targets">
    {/* Only handles horizontal overflow, not vertical */}
```

### Issue 3: Actual Sales Not Auto-Updating

**Location:** Backend - `/server/routes.ts` and database triggers

**Problem:** The app has TWO separate tables for sales data:

1. **`sales` table** (schema.ts line 63): Individual transaction logs
   - Each row = one sale/transaction
   - Has fields: `salesmanId`, `customerName`, `amount`, `date`, etc.
   - This is where users LOG individual transactions

2. **`monthly_sales_tracking` table** (schema.ts line 171): Monthly aggregated data
   - Each row = one customer's sales for one month
   - Has fields: `customerId`, `month`, `year`, `budget`, `actual`
   - The `actual` field is what displays on the dashboard

**The Core Problem:**
- User logs a sale in the `sales` table (e.g., via a "Log Sale" form)
- Dashboard shows data from `monthly_sales_tracking.actual` field
- There's NO automatic sync between these tables
- The `actual` field must be manually updated OR calculated via aggregation

**Why This Happens:**

**Option A: Manual Entry System (Current Implementation)**
The `monthly_sales_tracking.actual` field might be designed for manual entry, not auto-calculation. Users would need to:
1. Log individual sales in the `sales` table
2. Separately go to Monthly Sales Tracking page and manually enter totals

**Option B: Missing Trigger/Sync Logic**
There should be either:
- A database trigger that updates `monthly_sales_tracking.actual` when `sales` are inserted
- A backend service that aggregates sales periodically
- Frontend logic that recalculates on page load

**Current Dashboard Code:**
```typescript
// /client/src/pages/dashboard.tsx - Lines 120-128
const currentMonthSalesData = monthlySales.filter(
  s => s.month === currentMonth && 
       s.year === currentYear &&
       userCustomerIds.includes(s.customerId)
);

const currentMonthSales = currentMonthSalesData.reduce((total, sale) => {
  return total + (sale.actual ? Number(sale.actual) : 0);
}, 0);
```

This reads from `monthlySales` (which queries `monthly_sales_tracking` table), NOT from individual `sales` transactions.

**What Should Happen:**
Either:
1. Auto-aggregate `sales` into `monthly_sales_tracking.actual` after each transaction
2. Change dashboard to calculate directly from `sales` table instead
3. Add a "Sync" or "Refresh" button to recalculate monthly totals

### Fix for Issue 3: Auto-Sync Sales to Monthly Tracking

**CRITICAL DISCOVERY:** The `sales` table does NOT have a `customerId` field! It only has:
- `salesmanId` (who made the sale)
- `customerName` (text name, not ID reference)

The `monthly_sales_tracking` table requires `customerId`.

This means the current architecture has a **data model issue** where:
1. Individual sales are tracked by salesman, not by customer
2. Monthly tracking is tracked by customer
3. There's no direct link between the two tables

**Two Possible Solutions:**

---

**Solution A: Add customerId to Sales Table (Recommended for Long-term)**

This requires a database migration to add the relationship.

**Step 1: Add customerId column to sales table**

```typescript
// Create migration file: /server/migrations/add-customer-id-to-sales.ts

import { db } from "../db";
import { sql } from "drizzle-orm";

async function migrateAddCustomerIdToSales() {
  // Add customerId column
  await db.execute(sql`
    ALTER TABLE sales 
    ADD COLUMN customer_id VARCHAR REFERENCES customers(id);
  `);
  
  // Try to populate existing records by matching customer names
  await db.execute(sql`
    UPDATE sales s
    SET customer_id = c.id
    FROM customers c
    WHERE LOWER(TRIM(s.customer_name)) = LOWER(TRIM(c.name))
    LIMIT 1;
  `);
  
  console.log("Migration complete");
}

migrateAddCustomerIdToSales();
```

**Step 3: Add Sync Function**

```typescript
// In /server/storage.ts

async syncMonthlyActualSales(
  customerId: string, 
  month: number, 
  year: number
): Promise<void> {
  // Calculate total sales for this customer in this month
  const result = await db
    .select({
      total: sql<number>`COALESCE(SUM(CAST(${sales.baseCurrencyAmount} AS NUMERIC)), 0)`,
      currency: sql<string>`${sales.currency}`
    })
    .from(sales)
    .where(
      and(
        eq(sales.customerId, customerId),
        sql`EXTRACT(MONTH FROM ${sales.date}) = ${month}`,
        sql`EXTRACT(YEAR FROM ${sales.date}) = ${year}`
      )
    );

  const totalSales = result[0]?.total || 0;
  const currency = result[0]?.currency || 'USD';

  // Update or create monthly_sales_tracking record
  const existing = await db
    .select()
    .from(monthlySalesTracking)
    .where(
      and(
        eq(monthlySalesTracking.customerId, customerId),
        eq(monthlySalesTracking.month, month),
        eq(monthlySalesTracking.year, year)
      )
    )
    .limit(1);

  if (existing.length > 0) {
    await db
      .update(monthlySalesTracking)
      .set({ 
        actual: totalSales.toString(),
        actualCurrency: currency,
        actualBaseCurrencyAmount: totalSales.toString()
      })
      .where(eq(monthlySalesTracking.id, existing[0].id));
  } else {
    await db.insert(monthlySalesTracking).values({
      customerId,
      month,
      year,
      budget: "0",
      budgetCurrency: currency,
      budgetBaseCurrencyAmount: "0",
      actual: totalSales.toString(),
      actualCurrency: currency,
      actualBaseCurrencyAmount: totalSales.toString()
    });
  }
}
```

app.get("/api/sales/monthly-total", isAuthenticated, async (req, res) => {
  try {

    const year = parseInt(req.query.year as string);

      .select({

**SQL Fix for Issue 3 (One-time):**
```sql
-- Sync all monthly_sales_tracking.actual from sales table
UPDATE monthly_sales_tracking mst
SET actual = (
  SELECT COALESCE(SUM(amount::numeric), 0)
  FROM sales s
  WHERE s.customer_id = mst.customer_id
    AND EXTRACT(MONTH FROM s.date) = mst.month
    AND EXTRACT(YEAR FROM s.date) = mst.year
);
```
