Audit and fix all systems related to customer interactions, pipeline updates, and sales logging across the entire CustomerPulse CRM.

Current Problems:

Quarterly Targets cannot be added or saved to customer profiles.

Action Items cannot be added or updated.

Sales logging features are not working and do not save properly.

Pipeline drag-and-drop does not persist status changes.

There are likely additional interaction types that are failing due to shared underlying issues (API routes, validation, missing fields, incorrect schema, etc.).

Required Fixes:

1. Customer Interactions (General)

Audit all interaction types and ensure they fully work end-to-end:

Quarterly Targets

Action Items

Notes

Follow-ups

Tasks

Sales logs

Any other interaction stored on customer profiles

For each interaction type:

Ensure creation, update, delete, and display all function correctly.

Ensure the customer profile shows the newly added data immediately.

Fix any broken Zod validations, missing customerId fields, or incorrect TypeScript types.

Verify backend routes correctly write to the database.

2. Quarterly Targets

Fix the API calls responsible for creating and saving targets.

Ensure targets appear on the customer page, analytics page, and dashboards.

Validate that all required fields (e.g., quarter, year, revenue, comments) are properly mapped.

3. Action Items

Fix the “Add Action Item” workflow so items reliably save to the customer.

Ensure status updates (complete/incomplete) persist correctly.

Ensure they appear in the customer timeline and customer detail sidebar.

4. Sales Logging (NEW REQUIREMENT)

Sales entries currently do not save — fix the entire workflow:

Audit the sales logging UI form.

Fix the backend route that writes sales records.

Ensure sales entries include:

customerId

amount

date

notes

createdBy

Ensure sales logs appear:

on the customer profile

in the sales dashboard

in reports

Add validation to prevent partial or broken sales entries.

5. Pipeline Drag & Drop

Pipeline updates must:

Persist new stage/status to the database immediately.

Update the UI without requiring a manual refresh.

Validate that only allowed stages can be moved into each status.

Fix any missing API hooks or incorrect mutation calls.

Create a unified structure for all interactions:

customer_id
interaction_type
date / due_date
metadata (JSON)
created_by
timestamps

Ensure all features follow this model consistently to prevent future breakage.

7. Backend Tests

Add automated tests verifying:

Quarterly targets save, load, and update correctly.

Action items save, complete, delete, and appear in the timeline.

Pipeline drag-and-drop updates persist.

8. Goal

After completing this work:
