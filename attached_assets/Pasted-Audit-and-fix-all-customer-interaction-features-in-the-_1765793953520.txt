Audit and fix all customer-interaction features in the CRM.

**Problem:**
Several interaction types cannot currently be added or saved to customer profiles. For example:

* Quarterly Targets cannot be added or updated.
* Action Items cannot be added to a customer (button does nothing or returns an error).
* The drag-and-drop pipeline interaction on the Pipeline page is not working.
  Since multiple features are failing, there are likely additional interaction types or API routes that are also broken.

**Requirements:**

1. **Audit the entire customer interaction flow**, including:

   * Quarterly Targets (create, update, delete)
   * Action Items (create, update, complete, delete)
   * Pipeline drag-and-drop status changes
   * Notes, follow-ups, tasks, or any other interaction types stored on the customer record
   * Customer timeline/history feed
   * Related backend routes, mutations, and database writes

2. **Fix all issues preventing data from saving**, including:

   * Missing or incorrect API endpoints
   * Zod validation failures
   * Incorrect field mappings (customerId, interactionType, etc.)
   * Database constraint errors
   * Missing awaits or wrong mutation names/hooks in the frontend

3. **Standardize the data model** for interactions so all interaction types follow the same pattern:

   * customer_id
   * interaction_type
   * date or due_date
   * metadata (JSON)
   * created_by
   * timestamps

4. Ensure that **all interactions correctly appear on the customer profile**, in:

   * The customer detail page
   * Timeline/activity feed
   * Pipeline views
   * Dashboard summaries

5. Fix the **Pipeline drag-and-drop** so:

   * Items move between stages
   * New status is saved in the database
   * UI updates without requiring a refresh

6. Add **backend tests** confirming:

   * Quarterly targets save and load correctly
   * Action items save and load correctly
   * Pipeline status updates persist
   * Invalid data returns proper errors without breaking the UI

**Goal:**
Every interaction type must be fully functional and persist correctly. Users should be able to add Quarterly Targets, Action Items, Notes, Pipeline updates, and any other customer interactions without errors, and the timeline should reflect all updates.
