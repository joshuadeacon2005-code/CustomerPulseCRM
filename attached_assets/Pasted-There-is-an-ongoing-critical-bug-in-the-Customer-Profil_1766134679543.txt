There is an **ongoing critical bug** in the Customer Profile edit/save flow related to **date formatting**.

### Issue
When editing or saving a customer profile, the save fails with the error:

> **“expected date, received string”**

This is blocking customer data from being saved and has occurred multiple times, so a **proper root-cause fix is required**, not a workaround.

---

### What you need to do

1. **Identify the exact source of the mismatch**
   - Trace where date fields are:
     - entered in the frontend (inputs / forms)
     - transformed or serialized (e.g. JSON payload)
     - validated (Zod / schema / backend validation)
     - persisted (database schema)
   - Identify **which field(s)** are failing (e.g. `createdAt`, `lastContacted`, `followUpDate`, etc.).

2. **Fix the date handling correctly**
   - Ensure consistency between:
     - frontend input type
     - API payload
     - validation schema
     - database column type
   - Decide on **one correct format** (e.g. ISO string → Date object) and apply it end-to-end.
   - Do **not** rely on implicit casting or temporary coercion.

3. **Audit all customer date fields**
   - This bug may exist in more than one field.
   - Check **create**, **edit**, and **update** flows.
   - Ensure no date fields are silently converted to strings during edits.

4. **Test before completing**
   - Log in as a real user (admin or manager).
   - Open an existing customer.
   - Edit fields that include dates.
   - Save successfully **without errors**.
   - Refresh and confirm the data persists correctly.
   - Repeat with multiple customers.

5. **Only mark complete when verified**
   - Do not mark the task complete until:
     - The error no longer occurs
     - Customer profiles save reliably
     - Dates are stored and reloaded correctly

---

### Deliverable
- A confirmed fix with:
  - explanation of what caused the issue
  - what was changed
  - confirmation that the flow was tested end-to-end

This issue is **blocking normal CRM usage**, so accuracy and verification are required.
